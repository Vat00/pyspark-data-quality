# Локальный Data Quality конвейер на Apache PySpark

Проект реализует полноценный ETL-процесс (Извлечение, Трансформация, Загрузка) с автоматическим контролем качества данных (Data Quality).

## 🚀 Стек технологий
* **Язык**: Python 3.13
* **Big Data Фреймворк**: Apache PySpark (локальный кластер на базе Java 17)
* **Библиотеки**: Pandas, PyArrow, Faker (для генерации данных)
* **Форматы**: Вложенный JSON (сырые данные), CSV (аналитическое хранилище)

## 📋 Архитектура и этапы работы
1. `generator.py` — использует библиотеку `Faker` для генерации 100 реалистичных транзакций вымышленного банка. Скрипт намеренно вносит 15% брака (пустые ID, отрицательные суммы, системные ошибки).
2. `spark_processor.py` — инициализирует локальную сессию Spark, считывает сложный вложенный JSON, производит "уплощение" структуры (flattening) и выполняет проверку качества данных (Data Quality).
3. Валидные данные отделяются от брака и сохраняются силами Pandas в аналитическое хранилище `analytics_vault/`.

## 🛠 Инструкция по запуску
1. Установить зависимости:
   ```bash
   pip install faker pyspark pyarrow pandas
   ```
2. Запустить генератор сырых логов:
   ```bash
   python generator.py
   ```
3. Запустить Spark-обработчик:
   ```bash
   python spark_processor.py
   ```
